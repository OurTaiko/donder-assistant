# 太鼓达人难度计算器 (Taiko Rating Calculator)

一个基于 Electron 的桌面应用，用于批量计算太鼓达人谱面的多维度难度定数。

## 功能特点

✨ **四维度难度计算**
- 🏃 **体力定数** - 分析体力段分布和休息密度
- 🔀 **复合定数** - 计算复合难度占比
- 🎵 **节奏定数** - 评估节奏难度变化
- ⚡ **高速定数** - 测量高速打击难度

🚀 **核心功能**
- 📁 自动扫描 `data` 文件夹中的所有歌曲
- 🔄 批量计算所有歌曲的全部难度
- 📊 表格形式展示详细结果
- 🔍 实时搜索过滤功能
- 💾 一键导出 CSV 格式数据
- 🎨 现代化界面设计

## 安装和使用

### 前置要求
- Node.js 16.x 或更高版本
- npm 或 yarn

### 安装步骤

1. **安装依赖**
```bash
npm install
```

2. **启动应用**
```bash
npm start
```

3. **开发模式**（带开发者工具）
```bash
npm run dev
```

### 打包可执行文件

**打包所有平台**
```bash
npm run build
```

**仅打包 Windows**
```bash
npm run build:win
```
生成文件：
- `dist/太鼓难度计算器 Setup.exe` - 安装程序
- `dist/太鼓难度计算器.exe` - 便携版

**仅打包 macOS**
```bash
npm run build:mac
```
生成文件：`dist/太鼓难度计算器.dmg`

**仅打包 Linux**
```bash
npm run build:linux
```
生成文件：
- `dist/太鼓难度计算器.AppImage`
- `dist/太鼓难度计算器.deb`

**注意**：
- 打包后的文件位于 `dist` 文件夹
- Windows安装版支持自定义安装路径
- 便携版无需安装，直接运行
- data文件夹会自动包含在打包中

### 使用流程

1. **启动应用**
   - 运行 `npm start` 启动 Electron 应用

2. **开始计算**
   - 点击右上角的 "开始计算" 按钮
   - 应用会自动扫描 `data` 文件夹
   - 等待计算完成（进度条显示实时进度）

3. **查看结果**
   - 所有歌曲及其各难度的定数会显示在表格中
   - 每首歌的每个难度（oni/hard/normal/easy）占一行

4. **搜索过滤**
   - 在搜索框输入歌曲名称或分类名
   - 表格会实时过滤显示匹配结果

5. **导出数据**
   - 点击 "导出CSV" 按钮
   - 选择保存位置，保存为 CSV 文件

## 数据格式

### 输入格式
歌曲数据应放在 `data` 文件夹中，结构如下：

```
data/
└── [分类名称]/
    └── [歌曲名称]/
        └── [歌曲名称].json
```

JSON 文件格式示例：
```json

      ]

    "hard": { ... },
    "normal": { ... },
    "easy": { ... }
  }
```

### 输出格式
CSV 文件包含以下列：
- 分类
- 歌曲
- 难度
- 体力定数
- 复合定数
- 节奏定数
- 高速定数

## 算法说明

### 体力定数 (Stamina)
计算谱面的体力消耗，通过以下步骤：
1. 查找所有可能的体力段（累计 ≥ 20000 的连续区间）
2. 找到最优的两个不重叠体力段
3. 计算体力密度和休息密度
4. 输出加权平方平均值

### 复合定数 (Complex)
评估谱面的复合技巧难度：
1. 标记 L/R（左右手）
2. 计算换手标记
3. 计算同步标记
4. 基于标记变化计算基础系数
5. 应用大间隔修正
6. 输出复合难占比

### 节奏定数 (Rhythm)
分析谱面的节奏复杂度：
1. 分离左右手数组
2. 计算相邻间隔的比值
3. 应用大间隔修正系数
4. 应用馅蜜修正系数
5. 输出节奏难占比

### 高速定数 (Speed)
测量谱面的高速打击要求：
1. 计算相邻音符间隔和
2. 转换为速度值（1000/间隔）
3. 对速度值排名
4. 应用权重函数（p < 0.75 时权重为 0）
5. 输出加权平均值

## 常见问题

**Q: 为什么有些歌曲没有计算结果？**  
A: 可能是数据文件格式不正确或缺少必要的数据字段。

**Q: 计算很慢怎么办？**  
A: 计算是在主线程进行的，大量数据时会较慢。可以考虑优化算法或使用 Worker 线程。

**Q: 如何添加新的难度维度？**  
A: 在 `lib/` 中添加新的算法模块，然后在 `lib/calculator.js` 中集成。
